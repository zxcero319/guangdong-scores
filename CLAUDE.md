## ⚠️ Viewer 修改安全守则

**`guangdong_scores_viewer.html` 是单文件嵌入式应用，修改极易引入破坏性 bug。**

### 禁止行为
1. **禁止用 Python 字符串替换修改 `var ALL_DATA = [...]` 包裹部分** — 只替换 `[...]` 数组内容，不动前缀/后缀
2. **禁止在 renderTable 行模板中增删 `<td>` 时搞乱引号配对** — 每增删 `<td>` 确保单引号 `'` 数量平衡
3. **禁止在 exportCSV 的 `keys.map(function(k) { if... })` 块中增删 `if` 分支不加闭合花括号**
4. **禁止用 `html.replace(old, new)` 匹配可能出现在多处的 CSS/JS 片段** — 始终用唯一上下文锚定
5. **禁止用 `html[:b] + new_json + html[e-1:]` 拼接 JSON** — `new_json` 已含闭合 `]`，`html[e-1:]` 又以 `]` 开头，每次保存多一个 `]`，累积导致 JS 语法错误页面白屏。**正确写法: `html[:b] + new_json + html[e:]`**（`html[e:]` 是闭合 `]` 之后的内容）

### 每次修改后必做检查
```bash
python -c "
import json
with open('guangdong_scores_viewer.html','r',encoding='utf-8') as f: html=f.read()
idx=html.find('var ALL_DATA ='); b=html.find('[',idx)
d=0; e=b
for i in range(b,len(html)):
 if html[i]=='[': d+=1
 elif html[i]==']':
  d-=1
  if d==0: e=i+1; break
data=json.loads(html[b:e]); print(f'JSON: {len(data)} records')
js=html[html.find('<script>'):html.find('</script>')]
b_ok=js.count('{')==js.count('}'); p_ok=js.count('(')==js.count(')')
br_ok=js.count('[')==js.count(']'); sq_ok=js.count(\"'\")%2==0
print(f'Braces:{b_ok} Parens:{p_ok} Brackets:{br_ok} Quotes:{sq_ok}')
for fn in ['init','applyFilters','renderTable','exportCSV']:
 c=html.count(f'function {fn}(')
 if c!=1: print(f'WARN: {fn} count={c}')
print('ALL OK' if all([b_ok,p_ok,br_ok,sq_ok]) else 'ISSUES!')
"
```

## 数据提取规则（所有后续文件必须遵循）

### 字段定义

| 字段 | 含义 | 来源 |
|------|------|------|
| **WC** | 笔试人数 | 仅限笔试成绩表/笔试名单 |
| **IC** | 面试人数 | 体检名单总人数 / 面试名单 / 拟聘名单 / 签约名单 |
| **PR** | 计划招聘/实际录取人数 | 体检合格人数(优先) > 拟聘录用人数 > 签约人数 > 岗位表计划数 |
| **MS** | 进面分 — 最低笔试入围分数 | 仅限笔试成绩表中有分数数据的 |
| **TS** | 总成绩 — 综合/面试最低录取分 | 仅限成绩表中有分数数据的，取「体检=是」候选人的 min |

### 核心规则

- **WC**: 仅从笔试成绩表/笔试名单提取；拟聘/体检/签约名单不含笔试信息 → WC=0
- **IC**: 综合成绩表 → 全部参加面试人数（排除面试缺考）；体检名单/拟聘名单/签约名单总人数 → IC
- **PR**: 体检合格人数 > 拟聘录用人数 > 签约人数 > 岗位表计划数（仅参考）
- **TS**: `min(仅体检=是候选人的综合成绩)`，非全部候选人最低分
- **MS**: `min(笔试成绩)`，非综合成绩。**XLSX 综合成绩表中注意区分「笔试成绩」列与「综合成绩」列！**
- **WR/IR**: `WR = WC÷PR`, `IR = IC÷PR`（当 WC/IC>0 且 PR>0 时）

### 文件类型→字段映射

| 文件类型 | WC | IC | PR | MS | TS |
|----------|----|----|----|----|----|
| 笔试成绩表(含分数) | ✅ | — | — | ✅ | — |
| 面试名单 | — | ✅ | — | — | — |
| 体检结果名单 | — | ✅ | ✅ 合格 | — | — |
| 拟聘人员名单 | — | ✅ | ✅ | — | — |
| 签约名单 | — | ✅ | ✅ | — | — |
| 综合成绩表(含分数) | ✅ | ✅ | ✅ 体检=是 | ✅ | ✅ |
| 岗位需求表 | — | — | ⚠️ 计划数 | — | — |

### 去重/合并规则

| 类型 | 学校列特征 | 匹配方式 |
|------|-----------|----------|
| 通用修正 | 以「教育局」结尾 | 模糊匹配：district + position + year |
| 特定修正 | 真实学校名 | 精确匹配：district + school + position + year |

### 空学段推断优先级
1. 岗位名关键词：高中/初中/小学/幼儿/中职/特殊教育
2. 联合关键词：启智/启慧 → 特殊教育
3. 学校名关键词：职业中学→中职、幼儿园→幼儿
4. 同校已知学段 → 直接继承
5. 学科fallback：历史/物理/化学→中学；语文/数学(学校≠小学)→中学
